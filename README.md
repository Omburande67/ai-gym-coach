# AI Gym Coach

Privacy-first real-time workout recognition and smart fitness assistant platform.

## Overview

The AI Gym Coach analyzes user workouts through webcam video input, providing immediate feedback on exercise form, counting repetitions, and offering personalized coaching through an AI-powered conversational interface. The platform processes video frames locally to extract pose keypoints without storing or transmitting raw video data, ensuring user privacy while delivering professional-grade fitness guidance.

## Features

✅ **FULLY IMPLEMENTED - ALL CORE FEATURES COMPLETE**

- **Real-time Pose Detection**: Browser-based pose estimation using TensorFlow.js and MediaPipe BlazePose
- **Exercise Recognition**: Automatic detection of push-ups, squats, planks, and jumping jacks
- **Rep Counting**: Accurate repetition counting using biomechanical angle analysis
- **Form Analysis**: Real-time feedback on exercise form with corrective suggestions
- **AI Workout Plans**: Personalized 7-day workout plans generated by LLM based on user profile and history
- **AI Coach Chat**: Context-aware conversational AI assistant for fitness advice and motivation
- **Streak Tracking**: Automatic workout streak calculation with milestone notifications
- **User Profile & Statistics**: Comprehensive dashboard with progress tracking
- **Notification System**: Scheduled reminders and missed workout alerts
- **Privacy-First**: Video frames processed locally, only skeletal keypoints transmitted
- **Workout History**: Track progress over time with detailed statistics and summaries
- **Error Handling**: Global error boundary with user-friendly recovery options

## Getting Started

### Prerequisites

- Node.js 20+ and npm
- Python 3.11+
- Docker and Docker Compose
- PostgreSQL 16+
- Redis 7+

### Installation

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd ai-gym-coach
   ```

2. **Set up environment variables**
   
   Frontend:
   ```bash
   cd frontend
   cp .env.local.example .env.local
   # Edit .env.local with your configuration
   ```
   
   Backend:
   ```bash
   cd backend
   cp .env.example .env
   # Edit .env with your configuration (especially OPENAI_API_KEY)
   ```

3. **Start services with Docker Compose**
   ```bash
   docker-compose up -d
   ```
   
   This will start:
   - PostgreSQL on port 5432
   - Redis on port 6379
   - Backend API on port 8000
   - Frontend on port 3000

4. **Access the application**
   - Frontend: http://localhost:3000
   - Backend API: http://localhost:8000
   - API Documentation: http://localhost:8000/docs

## Architecture

The system follows a client-server architecture with privacy-first design:

1. **Video Processing (Browser)**: Video frames are captured and processed entirely in the browser using TensorFlow.js. Only skeletal keypoints are extracted and transmitted.

2. **Real-time Communication (WebSocket)**: Pose keypoints are sent to the backend via WebSocket for exercise recognition, rep counting, and form analysis.

3. **Backend Processing**: The backend analyzes pose data using rule-based biomechanical algorithms and provides real-time feedback.

4. **AI Features**: LLM integration provides personalized workout plans and conversational coaching.

5. **Data Persistence**: User profiles, workout history, and statistics are stored in PostgreSQL.

## Privacy & Security

- **No Video Storage**: Raw video frames are never stored or transmitted
- **Local Processing**: Pose detection happens entirely in the browser
- **Encrypted Passwords**: Bcrypt hashing for password storage
- **JWT Authentication**: Secure token-based authentication
- **HTTPS/WSS**: Encrypted communication in production
